Backup Lab#2
ในหัวข้อนี้จะคล้ายๆกับหัวข้อ web-backup ก่อนหน้า โดยในหัวข้อนี้จะยกตัวอย่าง temporary file ของ file ใด ๆ ที่จะถูกสร้างขึ้นเมื่อมีการใช้งาน vi, vim ซึ่งตามนิสัยปกติของ vim นั้นเมื่อมีการแก้ไขหรือเปิดไฟล์ใดๆ vim command จะสร้าง temporary file ของไฟล์ที่ถูกเปิดขึ้นมา ซึ่งไฟล์ดังกล่าวจะเป็น .swp เช่น หากเราเปิดไฟล์ test.txt เมื่อเปิดไฟล์ดังกล่าวด้วย vim จะมีการสร้างไฟล์ .test.txt.swp ขึ้นมา เป็นต้น ซึ่งหาก user ถูกตัด connection ในการใช้งานในขณะนั้นหรือก็คือโปรแกรม vim ตายก่อนที่จะทำการ save ไฟล์ .swp ก็จะยังคงถูกทิ้งค้างไว้ ซึ่งทำให้เราอาจสามารถเข้าถึงไฟล์ temporary file ดังกล่าวเพื่ออ่าน source code ของหน้าเพจนั้น ๆ ก็เป็นได้
